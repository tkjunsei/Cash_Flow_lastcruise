# 開発指示書「ログイン不要の共有型家計簿アプリ（Walica風）」

## 1. プロジェクトの概要
会員登録やログインを一切不要とし、発行された一意のURL（UUID）を知っているユーザーのみがアクセス・編集できる「共有型家計簿（支出管理）ウェブアプリケーション」を開発してください。
コンセプトは「Walica」のような手軽な共有体験です。

## 2. コア機能要件
1. **グループ作成機能**
   - トップページで「グループ名」を入力し、作成ボタンを押す。
   - バックエンドで推測不可能なUUID（v4）を生成し、それをキーにしたグループを作成。
   - 作成後、`/groups/[uuid]` の専用ページへリダイレクトする。
2. **支出の記録・編集・削除（CRUD）機能**
   - 専用ページ内で、日付、用途（タイトル）、金額、支払った人などを入力して保存できる。
   - リアルタイム、またはリロードなしでリストに反映されること。
3. **グラフ可視化機能**
   - 入力された支出データを元に、カテゴリ別または人別の支出割合を「円グラフ」や「棒グラフ」で視覚的に表示する。
4. **共有機能**
   - 現在のページのURLをワンクリックでクリップボードにコピーできるボタンを設置する。

## 3. 推奨技術スタック
- **フロントエンド**: Next.js (App Router), React, TypeScript
- **スタイリング**: Tailwind CSS, shadcn/ui (または類似のUIライブラリ)
- **グラフ描画**: Recharts (または Chart.js)
- **バックエンド/DB**: Supabase (PostgreSQL) または Firebase (Firestore)
- **ホスティング**: Vercel

## 4. データベース設計案（リレーショナルベースの場合）
以下の2つのテーブル（またはコレクション）を作成想定。

**Table: `groups`**
- `id`: uuid (Primary Key, 自動生成)
- `name`: text (グループ名)
- `created_at`: timestamp

**Table: `expenses`**
- `id`: uuid (Primary Key)
- `group_id`: uuid (Foreign Key -> groups.id)
- `title`: text (支出項目・用途)
- `amount`: integer (金額)
- `paid_by`: text (支払った人 - オプション)
- `date`: date (支払日)
- `created_at`: timestamp

## 5. セキュリティとUXの必須要件
- **認証の排除**: ユーザー認証（Auth）は実装しない。`group_id` (UUID) を知っているリクエストに対しては、対象グループのデータの読み書きを許可する（Row Level Security / セキュリティルールの設定に注意）。
- **クローラ対策**: 検索エンジンにインデックスされるとURLが漏洩するため、アプリ全体または `/groups/[id]` 以下のページに `<meta name="robots" content="noindex">` を必ず設定する。
- **履歴保存 (UX改善)**: ユーザーが一度アクセスしたグループのURLと名前をブラウザの `localStorage` に保存し、次回トップページにアクセスした際に「最近見たグループ」としてリスト表示する。
