# 【UI/UXデザイン変更指示】白基調・目に優しくわかりやすいデザイン

現在のアプリケーションのUIデザインを、以下の要件に従って「白を基調とした、目に優しく直感的にわかりやすい」モダンなデザインに改修してください。Tailwind CSSのユーティリティクラスを活用してください。

## 1. カラーパレットの指定
- **ベース背景色**: 真っ白（#FFFFFF）による画面のまぶしさを抑えるため、アプリ全体の背景にはごく薄いグレーやオフホワイト（例: `bg-slate-50` や `bg-gray-50`）を設定してください。
- **カード・コンテナ背景**: フォームやグラフ、リストなどを囲むコンテナの背景は純白（`bg-white`）にし、ベース背景とのコントラストでコンテンツのまとまりを明確にしてください。
- **テキストカラー**: 真っ黒（#000000）は目に負担がかかるため避け、メインテキストには濃いグレー（例: `text-slate-800`）、補足や日付などのサブテキストには少し薄いグレー（例: `text-slate-500`）を使用してください。
- **アクセントカラー（ボタンや強調箇所）**: 原色や蛍光色は避け、落ち着いたトーンで信頼感のある色（例: 柔らかいブルー `blue-500` や、穏やかなグリーン/ティール `teal-500`）を使用してください。

## 2. レイアウトと視覚的階層（わかりやすさの向上）
- **余白（Whitespace）**: ユーザーが情報を拾いやすいよう、要素同士の余白（`p-4`, `p-6`, `gap-4` など）をゆったりと取り、窮屈さをなくしてください。
- **影と立体感**: コンテナには控えめなドロップシャドウ（`shadow-sm` や `shadow`）を適用し、画面に奥行きを出して「どこに何の情報があるか」を直感的に区別できるようにしてください。
- **角丸（Border Radius）**: ボタンやカード、入力フィールドの角は少し丸め（`rounded-lg` や `rounded-xl`）、柔らかく親しみやすい印象を持たせてください。

## 3. 具体的なコンポーネントのスタイリング要件
- **入力フォーム**: 入力欄は薄い枠線（`border-slate-200`）を持ち、フォーカス時にアクセントカラーの枠線（例: `focus:ring-blue-500`）がハイライトされるようにして、入力状態をわかりやすくしてください。
- **ボタン**: メインのアクション（作成、保存など）はアクセントカラーの背景に白文字とし、ホバー時に少し色が変わる（例: `hover:bg-blue-600`）トランジション（`transition-colors`）をつけて、操作可能であることを明確にしてください。
- **支出リスト**: 各行の間に薄い区切り線（`divide-y divide-slate-100`）を入れるか、背景を交互に薄く変えて、項目と金額を目で追いやすくしてください。
- **グラフ**: グラフ（Rechartsなど）のカラーパレットも、目に優しいパステル調やくすみカラー（ソフトな青、緑、黄色などの組み合わせ）に設定してください。

上記の要件を満たすように、トップページ、入力フォーム、リスト、グラフ部分のUI（Tailwind CSSのクラス名）をアップデートしたコードを提示してください。
